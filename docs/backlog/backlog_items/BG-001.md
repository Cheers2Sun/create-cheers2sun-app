---

# BG-001 – Automatic Specification Initialization

## Category

CLI / Developer Experience

## Priority

Medium

## Target Sprint

Sprint 8

## Status

Open

---

## Problem

The generator currently requires a `cheers.yaml` specification before a project can be generated.

Running

```bash
create-cheers2sun-app mytravel
```

without a specification produces an error.

While technically correct, this creates unnecessary friction for first-time users.

---

## Current Behavior

When `cheers.yaml` is not found:

```
❌ Compiler failed.

Cheers specification not found:
/path/to/cheers.yaml
```

The user must already know that they should run:

```bash
create-cheers2sun-app init
```

or specify

```bash
create-cheers2sun-app mytravel \
    --spec path/to/cheers.yaml
```

---

## Proposed Enhancement

When no specification is found, the CLI should detect the missing file before compilation begins.

Example:

```bash
create-cheers2sun-app mytravel
```

Output:

```
No cheers.yaml found.

Create one from the default template? (Y/n)
```

If the user selects **Yes**:

```
✓ Created cheers.yaml

Generating project...
```

The generator then continues normally.

If the user selects **No**, the command exits gracefully.

---

## Alternative

Provide a command-line switch:

```bash
create-cheers2sun-app mytravel --init
```

which automatically creates the default specification and continues generation.

---

## Benefits

- Better first-run experience
- Fewer support questions
- Easier onboarding
- More intuitive CLI
- Reduces documentation dependency

---

## Design Considerations

The enhancement should preserve explicit behavior.

The CLI must never overwrite an existing `cheers.yaml` without confirmation.

If multiple specification templates are available in the future, the CLI may allow the user to choose one before generation begins.

---

## Dependencies

- Installation utility
- Init command
- CLI argument parser

---

## Acceptance Criteria

- Detect missing specification before compiler execution.
- Prompt user to create a default specification.
- Generate `cheers.yaml` using the default template.
- Continue project generation automatically.
- Never overwrite an existing specification without confirmation.

---

## Notes

This enhancement improves usability only.

It does not modify the Cheers Specification, Compiler, Generator, or Plugin Framework.
